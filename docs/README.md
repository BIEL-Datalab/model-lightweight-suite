# 文档目录

## 目录说明

本目录包含项目文档（架构约束、用户使用、开发者指南）。项目的“可运行命令与参数”以各入口脚本所在目录 README 为准：

- 根目录 README：环境隔离与快速开始
- quantization/README.md：量化入口与工具链说明
- evaluation/README.md：评估入口与输出说明
- resnet/readme.md：ResNet TensorRT PTQ 自包含工具包说明

## 子目录结构

```
docs/
├── architecture/                 # 架构文档
├── user_guide/                   # 用户指南
└── developer_guide/              # 开发者指南
```

## 子目录说明

### architecture/
包含项目的架构文档，如系统架构、模块设计、数据流等。

### user_guide/
包含用户指南，如安装指南、使用说明、常见问题等。

### developer_guide/
包含开发者指南，如开发环境搭建、代码风格、测试指南、贡献流程等。

## 文档格式

- 文档使用Markdown格式编写，便于阅读和维护
- 文档应包含清晰的标题和结构
- 代码示例应使用代码块包裹
- 图表应使用相对路径引用
- 文档应定期更新，保持与代码同步

## 文档编写规范

1. **清晰性**：文档应清晰易懂，避免使用过于专业的术语
2. **完整性**：文档应包含所有必要的信息，避免遗漏
3. **一致性**：文档的格式和风格应保持一致
4. **准确性**：文档应准确反映代码的实际功能
5. **时效性**：文档应及时更新，与代码保持同步

## 文档维护

- 新增功能时应同时更新相关文档
- 修改功能时应同步更新文档
- 定期检查文档的准确性和完整性
- 鼓励用户和开发者贡献文档
